**Q1. What is a nested condition?**
It is simply an `if` statement located inside another `if` statement.

**Q2. Why do we use nested if statements?**
They are used for multi-level decision making where one condition must pass before checking another.

**Q3. What happens if the outer condition is False?**
If the first (outer) condition is not met, the program ignores the inner condition entirely.

**Q4. Can nested conditions have else block?**
Yes, nested structures can include `else` or `elif` blocks at any level of the hierarchy.

**Q5. How do nested conditions help in real-world programs?**
They help model complex logic, like checking if a user is logged in AND has admin permissions.

Summary of learnings:
* Nested conditions are used to handle decisions that have multiple layers.
* The inner code block only executes if the outer condition evaluates to True.
* They are vital for creating detailed logic in applications like login systems or security checks.
